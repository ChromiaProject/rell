1. Library: JSON value retrieval methods and index operator

Strict item getter functions:

    json.get(index: integer): json

        Get the element at the specified index in a JSON array, as a JSON value. Throws an exception if the target JSON
        value is not an array, or if the index is out of bounds.

    json.get(key: text): json

        Get the element with the specified key in a JSON object (dictionary). Throws an exception if the target JSON
        value is not an object (dictionary), or if the given key does not exist.

Nullable getter functions:

    json.get_or_null(integer): json?
    json.get_or_null(text): json?

        These are the same as the above get() functions, but return null in cases where the get() variant would throw an
        exception.

Strict conversion functions:

    json.as_integer(): integer
    json.as_text(): text
    json.as_boolean(): boolean

        Convert a JSON value to a Rell value. Throws an exception if the JSON value is not of the correct type; e.g.
        calling as_text() on a JSON integer will throw an exception.

Nullable conversion functions:

    json.as_integer_or_null(): integer?
    json.as_text_or_null(): text?
    json.as_boolean_or_null(): boolean?

        These are the same as the above as_integer(), as_text() and as_boolean() conversion functions, but return null
        in cases where the standard variant would throw an exception.

Functions to check the type of a JSON value:

    json.is_object(): boolean
    json.is_array(): boolean
    json.is_text(): boolean
    json.is_integer(): boolean
    json.is_boolean(): boolean
    json.is_null(): boolean

        Each return true if the JSON value is of the type in the method's name, or false otherwise.

Other functions:

    json.size(): integer

        Get the size of a JSON value; i.e. the number of elements in a JSON array, or the number of key-value pairs in a
        JSON object. Throws an exception if the JSON value is not an array or an object.

    json.keys(): set<text>

        Get the keys of a JSON object. Throws an exception if the JSON value is not an object.

Index operator []

    The JSON type now supports an index operator [], equivalent to the above get() methods.

    Examples:
    json('{"x": 1, "y": 2}')['x'] returns json('1')
    json('["a", "b", "c"]')[1] returns json("b")

Full example showing combined usage of the new functions and operators to navigate JSON trees:

    Let the variable user_data be of type json with the following data:

        {
            "user": {
                "id": 123,
                "profile": {
                    "nickname": null,
                    "preferences": { "theme": "dark", "volume": 5 }
                },
                "age": 30
            },
            "sessions": [
                { "id": "s1", "meta": { "started": "2025-10-10T09:00:00Z", "ip": "10.0.0.1" } },
                { "id": "s2", "meta": { "started": "2025-10-09T18:12:00Z" } }
            ]
        }

    Then we can write the following:

        user_data['sessions'][0]['id'].as_text() // returns 's1'
        user_data['user']['profile']['preferences']['volume'].as_integer() // returns 5

2. Library: function text.regex_replace(regex: text, replacement: text)

Returns the text obtained by replacing all subsequences in this text which match the given regex, with the
replacement text. Supports named and numbered group references in the replacement text.

Examples:
'aaaaacaaaaabaaaaacaaaaa'.regex_replace('(b|c)', 'd') // returns 'aaaaadaaaaadaaaaadaaaaa'.
'aaaaacaaaaabaaaaacaaaaa'.regex_replace('a+', '') // returns 'cbc'.
'johnsmith@chromaway.com'.regex_replace('[a-z]+', '0') // returns '0@0.0'.
'John Doe'.regex_replace('(\\w+) (\\w+)', '$2, $1') returns 'Doe, John'.
'2025-10-27'.regex_replace('(?<y>\\d{4})-(?<m>\\d{2})-(?<d>\\d{2})', '${d}/${m}/${y}') returns '27/10/2025'.

3. Library: function try_call_catch() and type try_call_result<T>

A new function try_call_catch() provides safe execution of code that may fail with require() exceptions, capturing
failure details in a structured result.

(a) Global function:

    try_call_catch<T>(f: () -> T): try_call_result<T>

    Executes the provided function f and returns a structured result encapsulating the outcome. If the function executes
    successfully, the result contains the return value. If the function fails with a require() exception, the result
    captures the error message. All other exceptions are re-thrown.

    Database changes made during execution are automatically rolled back if the function fails with a require()
    exception, using database savepoints to ensure transactional integrity.


(b) Type try_call_result<T> - properties:
    is_error: boolean - returns true if the function failed with a require() exception, false if it succeeded.
    value: T - returns the function's result value if successful. Throws an exception if the function failed.
    value_or_null: T? - returns the function's result value if successful, or null if the function failed.
    require_message_or_null: text? - returns the require() exception message if the function failed, null if succeeded.

Code examples:

function get_user(id: integer): text {
    require(id > 0, "Invalid user ID");
    return "User_" + id;
}

val result1 = try_call_catch(get_user(123, *));
// result1.is_error == false
// result1.value == "User_123"
// result1.value_or_null == "User_123"
// result1.require_message_or_null == null

val result2 = try_call_catch(get_user(-1, *));
// result2.is_error == true
// result2.value throws exception
// result2.value_or_null == null
// result2.require_message_or_null == "Invalid user ID"

// Conditional error handling:

val result = try_call_catch(get_user(user_id, *));

if (result.is_error) {
    log("Failed: " + result.require_message_or_null!!);
} else {
    process_user(result.value);
}
